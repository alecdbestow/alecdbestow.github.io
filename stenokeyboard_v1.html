Problem Statement

Stenography keyboards can type with a very high words per minute, reaching above 200wpm. This makes them very attractive for typing. However, the modern keyboard versions require software to run on the computer. This results in you needing two keyboards, one for stenography, and one for when the software is not running, ie when your computer has just turned on. In addition, steno keyboards aren't very good at typing code are other complex text that requires symbols and words that don't appear in the normal dictionary.

Design Requirements

  • The keyboard should be able to perform all the functions of normal keyboard
  • The steno dictionary should be easily editable, both in bulk and in use
  • There should be no noticeable latency
  • It should be appropriate to use in the workplace
  • It should perform the steno function well
  • It should perform the normal keyboard function well

Alternatives

  • The Characorder doens't have an open source dictionary and only has a web interface for interacting with it. In addition, the stenography keys aren't vertically alligned and they keyboard is inclined. This makes it harder to press more than one at key at a time, as required by stenography.
  • The polyglot is less fleshed out as a keyboard and has too few keys to be easy to use in day to day use. Otherwise is pretty much ideal with the embedded steno version.

Development and Design

The initial backplate was large and made from mild steel. The "kerf" was too wide, meaning the switches didn't fit into place snugly, leading to slightly rotated keys and an insecure plate.

After some testing the size of the keyboard was too large to be useful. As it needed to be made again due to the poorly fitting, alternative layots were investigated.

After some investigation, a very simple ortholinear layout was more appropriate as it's much easier to buy a set of keys and no stabilisers are necessary.

The design was then lasercut from 1.4mm aluminium using the metal laser cutter at UNSW Kirby Makerspace. This meant the kerf issue could be resolved as several test squares could be cut and tested before production to find the optimal size of hole to fit the switches.

EasyEda makes designing PCBs very easy.

The patterns of the buttons were put on the wrong side. Not wanting to re-order, an attempt at fixing was made. It turns out this could be fixed without much issue. Materials

  • XDAL White keycap set

  • 70 Gateron white 38g linear switches
  • 1 PCB
  • Block of Redgum
  • 1.4mm aluminium plate
  • 1 Sparkfun Pro Micro rp2040
  • 1 USB-C extension
